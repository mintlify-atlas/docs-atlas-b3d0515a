## Overview

Categories and tags are powerful organizational tools that help you understand your spending and income patterns. While they serve similar purposes, they work differently and excel in different scenarios.

## Categories

Categories classify transactions by type or purpose:
- **Groceries**: Food shopping
- **Utilities**: Electric, water, gas bills  
- **Entertainment**: Movies, games, concerts
- **Transportation**: Gas, public transit, parking
- **Healthcare**: Medical expenses, prescriptions

<Note>
Categories answer the question: "What was this transaction for?"
</Note>

## Tags

Tags are flexible labels you can apply to transactions:
- `#vacation-2024`: Trip-related expenses
- `#shared`: Split expenses with roommates
- `#reimbursable`: Expenses you'll be reimbursed for
- `#work`: Business-related purchases
- `#urgent`: Priority expenses

<Note>
Tags answer questions like: "Which project is this for?" or "Who does this involve?"
</Note>

## Working with Categories

### Creating Categories

<Steps>
  <Step title="Navigate to Categories">
    Go to **Categories** in the main navigation.
  </Step>
  
  <Step title="Create New Category">
    Click **Create New Category**.
  </Step>
  
  <Step title="Name the Category">
    Enter a clear, descriptive name ("Home Maintenance" or "Pet Care").
  </Step>
  
  <Step title="Add Notes (Optional)">
    Describe what belongs in this category.
  </Step>
  
  <Step title="Save">
    Click **Save** to create the category.
  </Step>
</Steps>

  Start with 10-15 major categories. You can always:
  - Add more as needs emerge
  - Merge categories that aren't useful
  - Split categories that are too broad
  
  Too many categories = overwhelming
  Too few categories = not enough detail
</Accordion>

### Assigning Categories

**During Transaction Creation:**
1. Fill in transaction details
2. Select category from dropdown
3. Save transaction

**After Transaction Creation:**
1. Open transaction
2. Click **Edit**
3. Select or change category
4. Save

**Automatically with Rules:**
Use [Rules](/features/rules) to auto-categorize:
- "Whole Foods" → "Groceries"
- "Shell Station" → "Transportation"
- "Netflix" → "Entertainment"

### Editing Categories

To modify a category:
1. Go to **Categories**
2. Click on the category
3. Click **Edit**
4. Update name or notes
5. Save

<Warning>
Renaming a category updates all transactions using that category. Past and future transactions will show the new name.
</Warning>

### Deleting Categories

<Warning>
You cannot delete categories with transactions assigned to them. First:
1. Reassign all transactions to different categories
2. Or merge this category with another
3. Then delete the empty category
</Warning>

### Merging Categories

Combine duplicate or similar categories:
1. Identify categories to merge (e.g., "Food" and "Groceries")
2. Open one category
3. Click **Merge**
4. Select which category to merge into
5. Confirm merge

All transactions from merged categories combine into one.

### No Category

Transactions without categories:
- View via **No Category** page
- Identify uncategorized spending
- Bulk assign categories

<Tip>
Run a "No Category" report monthly to catch uncategorized transactions. Proper categorization improves report accuracy.
</Tip>

## Working with Tags

### Creating Tags

Tags are created automatically when you use them:
1. When creating/editing a transaction
2. Type tag name in the tags field
3. Tags are created if they don't exist

**Or create explicitly:**
1. Go to **Tags** page
2. Click **Create New Tag**
3. Enter tag name and optional details
4. Save

### Tag Naming Conventions

<Accordion title="Best practices for tag names">
  - **Use prefixes**: #project-, #category-, #person-
  - **Be consistent**: Always "#vacation" not "#trip" sometimes
  - **Avoid spaces**: Use hyphens (#road-trip not #road trip)
  - **Keep it short**: Long tags are hard to use
  - **Use lowercase**: Easier to type and search
</Accordion>

### Applying Tags

**Single Tag:**
1. In transaction form, click tags field
2. Type tag name
3. Select from dropdown or create new

**Multiple Tags:**
1. Add first tag
2. Press comma or Enter
3. Add next tag
4. Repeat as needed

**Example:**
`#vacation-2024, #shared, #reimbursable`

### Tag Details

Tags can include additional metadata:
- **Date**: Associate tag with specific time
- **Description**: Explain tag's purpose
- **Location**: Geographic tags
- **Notes**: Additional context

<Accordion title="Advanced: Tag dates and locations">
  Tags can have dates and locations for special tracking:
  
  **Tag Date**: 
  - Mark when an event occurred
  - Track project timelines
  - Filter by tag date ranges
  
  **Tag Location**:
  - Geographic tracking
  - Map view of tagged transactions
  - Analyze spending by location
</Accordion>

### Editing Tags

1. Go to **Tags** page
2. Click on the tag
3. Edit details
4. Save changes

### Deleting Tags

To remove a tag:
1. Open tag details
2. Click **Delete**
3. Confirm deletion

<Note>
Deleting a tag removes it from all transactions. The transactions remain, but the tag association is lost.
</Note>

## Organizing Strategies

### The Hybrid Approach

Combine categories and tags effectively:

**Categories for**: What the expense is
- Groceries, Transportation, Entertainment

**Tags for**: Context and special tracking
- #vacation, #work, #shared, #reimbursable

**Example Transaction:**
- Description: "Dinner at Italian restaurant"
- Category: "Dining Out"
- Tags: #vacation-2024, #anniversary

### Project Tracking

Track project expenses:

<Steps>
  <Step title="Create Project Tag">
    Create #project-name (e.g., #bathroom-renovation)
  </Step>
  
  <Step title="Tag All Related Transactions">
    Apply tag to all project expenses
  </Step>
  
  <Step title="Generate Reports">
    Filter reports by tag to see project totals
  </Step>
  
  <Step title="Track Progress">
    Monitor spending against project budget
  </Step>
</Steps>

### Shared Expense Tracking

Manage split costs:

1. **Tag shared expenses**: #shared or #shared-with-name
2. **Record full amount**: Enter what you paid
3. **Track reimbursements**: When roommate pays you back
4. **Run reports**: See total owed/owing by tag

### Reimbursable Expenses

Track business or reimbursable expenses:

<Steps>
  <Step title="Tag When Created">
    Mark transaction with #reimbursable
  </Step>
  
  <Step title="Track in Reports">
    View all reimbursable expenses
  </Step>
  
  <Step title="Submit for Reimbursement">
    Use report as expense report
  </Step>
  
  <Step title="Record Reimbursement">
    Create deposit when reimbursed
  </Step>
  
  <Step title="Remove Tag">
    Edit original transactions, remove #reimbursable
  </Step>
</Steps>

## Category and Tag Reports

### Category Reports

Analyze spending by category:
- **Spending per category**: Which categories consume most money
- **Income per category**: Where income comes from
- **Month-over-month**: Category trends
- **Category budgets**: Spending vs. budget by category

### Tag Reports

Filter transactions by tags:
- **Tag summary**: Total spending per tag
- **Multi-tag reports**: Transactions with multiple tags
- **Tag timelines**: Spending over time for tagged items
- **Tag comparison**: Compare different tags

### Advanced Filtering

Combine categories and tags:
- "Groceries" category with #vacation tag
- "Transportation" category without #work tag
- Multiple categories, specific tags

## Best Practices

<Accordion title="Categorize Every Transaction">
  Make categorizing a habit:
  - Assign categories when creating transactions
  - Use rules to auto-categorize
  - Review uncategorized transactions weekly
  - Accurate categories = better reports
</Accordion>

<Accordion title="Keep Categories Simple">
  Don't over-complicate:
  - Start broad, add detail only if needed
  - Merge similar categories
  - 15-20 categories is usually enough
  - More categories ≠ better insights
</Accordion>

<Accordion title="Use Tags Sparingly">
  Tags should add value:
  - Only tag when there's a reason
  - Don't tag everything
  - Focus on tracking specific projects or contexts
  - Review and clean up unused tags
</Accordion>

<Accordion title="Be Consistent">
  Consistency improves tracking:
  - Use same category names always
  - Don't switch between similar categories
  - Standardize tag naming
  - Document your category structure
</Accordion>

<Accordion title="Review and Refine">
  Adjust over time:
  - Monthly: Check for uncategorized transactions
  - Quarterly: Review category usefulness
  - Yearly: Merge or split categories as needed
  - Delete unused tags
</Accordion>

## Troubleshooting

<Accordion title="Can't delete a category">
  Categories with transactions can't be deleted. Options:
  - Merge it with another category
  - Reassign all transactions to other categories first
  - Keep it but make it inactive (if supported)
</Accordion>

<Accordion title="Too many categories">
  Simplify your structure:
  1. Identify rarely-used categories
  2. Merge similar categories together
  3. Move specific categories to tags
  4. Focus on categories you actually use for decisions
</Accordion>

<Accordion title="Reports don't show expected data">
  Check:
  - Transactions are properly categorized
  - Date range includes the transactions
  - Category names are spelled consistently
  - No duplicate categories with similar names
</Accordion>

<Accordion title="Tag not appearing in lists">
  Verify:
  - Tag was saved to transactions
  - Spelling is consistent
  - Browser cache is clear
  - Tag is still assigned to active transactions
</Accordion>
