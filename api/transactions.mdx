## Overview

The Transactions API allows you to create and manage transaction groups. Each transaction group can contain multiple transaction splits, supporting complex financial scenarios.

## List All Transactions

<CodeGroup>
```bash cURL
curl -X GET "https://demo.firefly-iii.org/api/v1/transactions" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Accept: application/json"
```
</CodeGroup>

**Endpoint:** `GET /v1/transactions`

## Create Transaction

<CodeGroup>
```bash cURL
curl -X POST "https://demo.firefly-iii.org/api/v1/transactions" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "group_title": "Weekly groceries",
    "error_if_duplicate_hash": false,
    "apply_rules": true,
    "fire_webhooks": true,
    "transactions": [
      {
        "type": "withdrawal",
        "date": "2024-01-15T10:30:00Z",
        "amount": "125.50",
        "description": "Grocery shopping",
        "source_id": "1",
        "destination_name": "Supermarket",
        "category_name": "Groceries",
        "budget_id": "3",
        "tags": ["food", "essential"]
      }
    ]
  }'
```
</CodeGroup>

**Endpoint:** `POST /v1/transactions`

### Request Body

<ParamField body="group_title" type="string">
  Optional title for the transaction group (max 1000 characters)
</ParamField>

<ParamField body="error_if_duplicate_hash" type="boolean" default="false">

<ParamField body="fire_webhooks" type="boolean" default="true">

</ParamField>

<ParamField body="transactions" type="array" required>

</ParamField>

<Info>
**Transaction Types:**
- **withdrawal**: Money leaves your asset accounts
- **deposit**: Money enters your asset accounts
- **transfer**: Money moves between asset accounts
- **opening-balance**: Set initial account balance
- **reconciliation**: Balance adjustment
</Info>

## Get Transaction by ID

<CodeGroup>
```bash cURL
curl -X GET "https://demo.firefly-iii.org/api/v1/transactions/{id}" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Accept: application/json"
```
</CodeGroup>

**Endpoint:** `GET /v1/transactions/{id}`

Returns the same response structure as the List endpoint.

## Update Transaction

<CodeGroup>
```bash cURL
curl -X PUT "https://demo.firefly-iii.org/api/v1/transactions/{id}" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "transactions": [
      {
        "description": "Updated description",
        "amount": "150.00",
        "category_name": "Food & Dining"
      }
    ]
  }'
```
</CodeGroup>

**Endpoint:** `PUT /v1/transactions/{id}`

Accepts the same parameters as Create Transaction. All fields are optional.

## Delete Transaction

<CodeGroup>
```bash cURL
curl -X DELETE "https://demo.firefly-iii.org/api/v1/transactions/{id}" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Accept: application/json"
```
</CodeGroup>

**Endpoint:** `DELETE /v1/transactions/{id}`

Deletes the entire transaction group and all splits.

<Warning>
Deleting a transaction cannot be undone. All associated splits will be permanently removed.
</Warning>

## Related Endpoints

### Get Transaction Attachments

**Endpoint:** `GET /v1/transactions/{id}/attachments`

Returns all attachments for this transaction.

### Get Transaction Piggy Bank Events

**Endpoint:** `GET /v1/transactions/{id}/piggy-bank-events`

Returns piggy bank events triggered by this transaction.
